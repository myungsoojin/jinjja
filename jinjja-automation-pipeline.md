# jinjja.com — 자동화 파이프라인 설계 v1.0

## 전체 흐름 요약

```
[1. 키워드 수집] → [2. 장소 데이터 수집] → [3. AI 콘텐츠 생성] → [4. 품질 검수] → [5. 발행]
     자동              자동                  자동              자동/수빈          자동
```

각 단계를 하나씩 뜯어보자.

---

## STEP 1: 키워드 수집 (자동, 매주 1회)

### 뭘 하는 단계?
사람들이 지금 뭘 검색하는지 자동으로 수집.
이게 있어야 "아무도 안 찾는 글" 대신 "다들 찾는데 제대로 된 답이 없는 글"을 만들 수 있음.

### 수집 소스 3가지

**① 네이버 자동완성 스크래핑**
- 입력: 지역명 (성수동, 홍대, 강남, 부산, 제주...)
- 수집: 자동완성 결과 (성수동 카페, 성수동 맛집, 성수동 데이트...)
- 입력: 상황 키워드 (비오는날, 데이트, 혼밥, 부모님...)
- 수집: 자동완성 결과 (비오는날 데이트, 비오는날 카페, 비오는날 실내...)
- 조합: 지역 × 상황 키워드 자동 생성
  예) "홍대 비오는날 카페", "강남 소개팅 레스토랑"

**② 구글 트렌드 / 네이버 데이터랩**
- 시즌별 급상승 키워드 자동 감지
- 예: 3월이면 "벚꽃 명소" 급상승 → 자동으로 벚꽃 관련 페이지 생성 트리거
- 예: 12월이면 "크리스마스 데이트" → 자동 트리거

**③ 경쟁 사이트 상위 노출 키워드 (월 1회)**
- 다이닝코드, 블루리본, 망고플레이트, KKday 블로그의 상위 페이지 분석
- 그 중 "상황별" 키워드인데 우리가 아직 없는 것 추출

### 결과물
→ 키워드 DB에 저장 (예시):
| 키워드 | 월 검색량 추정 | 경쟁도 | 상황 태그 | 지역 태그 | 우선순위 |
|--------|---------------|--------|-----------|-----------|----------|
| 성수동 데이트 코스 | 12,000 | 중 | 데이트 | 성수동 | ★★★ |
| 비오는날 실내 데이트 서울 | 8,000 | 낮음 | 비오는날 | 서울 | ★★★ |
| 홍대 혼밥 맛집 | 6,500 | 중 | 혼자 | 홍대 | ★★☆ |
| 2026 벚꽃 명소 | 40,000 | 높음 | 시즌 | 전국 | ★★★ |

## STEP 2: 장소 데이터 수집 (자동, 키워드별)

### 뭘 하는 단계?
키워드에 맞는 실제 장소 데이터를 수집.
"성수동 데이트 코스"면 → 성수동의 데이트에 적합한 카페/식당/명소를 긁어옴.

### 수집 소스

**① Google Places API**
- 키워드 기반 장소 검색
- 수집 데이터: 장소명, 주소, 평점, 리뷰 수, 카테고리, 영업시간, 가격대, 사진 URL
- 비용: 월 $200 무료 크레딧으로 시작 가능

**② 네이버 플레이스 스크래핑**
- 한국어 리뷰, 방문자 리뷰 수, 블로그 리뷰 수
- 예약 가능 여부, 메뉴 정보
- 네이버 평점 (구글과 다를 수 있음 → 둘 다 보여주면 차별화)

**③ 카카오맵 API**
- 장소 검색 + 좌표
- 지도 임베드용

**④ 한국관광공사 TourAPI (무료)**
- 관광지, 축제, 행사 정보
- 영문 데이터 포함 → 영어 버전 자동 생성에 활용

**⑤ 기상청 API (무료)**
- 지역별 날씨 데이터
- "비오는날" 카테고리와 연동: 오늘 비 오는 지역 → 해당 지역 비오는날 페이지 자동 부스트

### 데이터 가공 로직

수집된 raw 데이터를 jinjja.com용으로 가공:

```
입력: Google Places + 네이버 플레이스 raw 데이터

가공 로직:
1. 중복 제거 (같은 장소 다른 소스 → 병합)
2. 상황 태그 자동 분류:
   - 리뷰에 "혼자", "혼밥", "1인" 키워드 → [혼자갈때] 태그
   - 리뷰에 "분위기", "데이트", "커플" → [데이트] 태그
   - 리뷰에 "아이", "유아", "키즈" → [아이와함께] 태그
   - 리뷰에 "노트북", "작업", "공부" → [작업하기좋은] 태그
   - 영업시간 22시 이후 → [야간] 태그
   - 가격대 낮음 → [가성비] 태그
   - 실내/실외 구분 → [비오는날] 태그 여부
3. 신뢰도 점수 계산:
   - (구글 평점 × 0.4) + (네이버 평점 × 0.4) + (리뷰 수 log × 0.2)
   - 4.0 이상 + 리뷰 100개 이상 = 우선 노출
4. "진짜" 필터:
   - 평점 높은데 리뷰 수 적음 → 의심 표시
   - 최근 6개월 리뷰 비율 확인 → 최근 리뷰 없으면 경고
```

### 결과물
→ 장소 DB에 저장 (예시):
| 장소명 | 지역 | 카테고리 | 구글평점 | 네이버평점 | 리뷰수 | 상황태그 | 가격대 | 신뢰도 |
|--------|------|----------|---------|-----------|--------|---------|--------|--------|
| 어니언 성수 | 성수 | 카페 | 4.3 | 4.5 | 3,200 | 데이트,작업 | ₩₩ | 높음 |
| 도산분식 | 강남 | 한식 | 4.1 | 4.4 | 850 | 혼자,가성비 | ₩ | 높음 |

## STEP 3: AI 콘텐츠 생성 (자동)

### 뭘 하는 단계?
수집된 키워드 + 장소 데이터를 jinjja 톤의 글로 변환.
여기가 진짜 핵심. 그냥 AI 글이 아니라 "jinjja 글"이 되어야 함.

### 콘텐츠 생성 프롬프트 시스템

**시스템 프롬프트 (모든 글에 공통 적용)**:
```
너는 jinjja.com의 에디터야.
jinjja.com은 한국의 맛집/카페/놀거리를 상황별로 큐레이션하는 사이트.

톤 규칙:
- 솔직하게. 별로면 별로라고 써.
- 재치있게. 정보인데 읽는 맛이 있어야 해.
- 실용적으로. 예쁜 말보다 진짜 도움 되는 정보.
- "~합니다" 관광 가이드체 절대 금지.
- "~임", "~거든", "~인데" 같은 자연스러운 구어체 사용.
- 따옴표 사용 금지.
- 한 장소당 반드시 포함: 솔직 한줄평 + 꿀팁 1개 + 누구에게 추천/비추인지

나쁜 예: 어니언 성수는 성수동을 대표하는 인기 카페입니다.
좋은 예: 어니언 성수, 맨날 줄 서지만 2층 창가석 잡으면 웨이팅 값을 함. 근데 빵은 솔직히 비싸고 양 적음. 커피 + 분위기로 먹는 곳.
```

### 페이지 유형별 생성 템플릿

**유형 A: 상황×지역 페이지** (Tier 1, 자동)
```
페이지: /상황별/데이트/서울-성수동/

입력 데이터:
- 상황: 데이트
- 지역: 성수동
- 해당 장소 리스트: [장소 DB에서 성수동 + 데이트 태그 필터]

생성 구조:
1. 인트로 (2~3줄)
   → 성수동이 데이트 코스로 어떤 곳인지 솔직하게
   → 예: "성수동 데이트, 일단 골목골목 돌아다니는 것 자체가 코스임. 
          굳이 계획 안 세워도 걷다 보면 뭔가 나옴. 근데 주말엔 사람 미침."

2. 추천 장소 리스트 (5~10곳)
   각 장소마다:
   - 장소명 + 카테고리
   - 솔직 한줄평 (AI 생성, jinjja 톤)
   - 구글/네이버 평점 (둘 다 표시)
   - 가격대 (₩/₩₩/₩₩₩)
   - 꿀팁 1개
   - 추천 대상 / 비추 대상
   - 지도 위치 (카카오맵 임베드)

3. 이 근처 다른 상황 (내부 링크)
   → 성수동 혼자갈때, 성수동 카페, 성수동 야간
   → /지역/서울/성수동/ (지역 허브)

4. 비슷한 상황 다른 지역 (내부 링크)
   → 홍대 데이트, 한남동 데이트, 을지로 데이트
```

**유형 B: 큐레이션 리스트** (Tier 2, AI+수빈 감수)
```
페이지: /먹을곳/카페/서울-노트북-작업-카페-BEST/

생성 구조:
1. 인트로: 왜 이 리스트를 만들었는지 (1~2줄)
2. 선정 기준 명시 (콘센트 유무, 와이파이 속도, 자리 넓이 등)
3. 리스트 (7~15곳), 순위 매기지 않음 (지역별로 분류)
4. 각 장소: 한줄평 + 데이터 + 꿀팁
5. 총평: 결론적으로 이러이러한 곳이 좋았음
6. 내부 링크 연결
```

**유형 C: 솔직리뷰** (Tier 3, 수빈 작성)
```
페이지: /솔직리뷰/과대평가-주의보/성수동-카페-3곳/

수빈 입력: 사진 3~5장 + 메모 (장소명, 먹은것, 느낌, 가격)
AI 가공: 수빈 메모 → jinjja 톤 초안 생성
수빈 감수: 10~15분 수정

구조:
1. 왜 이 글을 쓰게 됐는지 (1~2줄)
2. 각 장소 리뷰
   - 기대 vs 현실
   - 수빈의 솔직 평가
   - 실제 사진
   - 그래서 다시 갈 거냐? (Yes/No + 이유)
3. 결론
```

### SEO 자동 생성 요소

모든 페이지에 자동 적용:
```
- title: {상황} {지역} 추천 | 진짜만 알려줌 - jinjja
  예: "성수동 데이트 코스 추천 | 진짜만 알려줌 - jinjja"

- meta description: 현지인이 솔직하게 알려주는 {지역} {상황} 추천. 
  별로면 별로라고 말하는 jinjja에서 진짜 좋은 곳만 확인하세요.

- OG image: 자동 생성 (장소 사진 + jinjja 로고 오버레이)

- Schema markup: Article + LocalBusiness (각 장소)

- hreflang: ko ↔ en 연결 (영어 버전 있는 경우)
```

## STEP 4: 품질 검수 (자동 + 수빈)

### 자동 검수 (모든 페이지)
```
체크리스트:
□ 글 길이 500자 이상
□ 장소 최소 5곳 이상 포함
□ 각 장소에 한줄평 있음
□ 내부 링크 3개 이상
□ 이미지 1개 이상
□ title, description 생성 완료
□ 중복 콘텐츠 체크 (기존 페이지와 30% 이상 겹치면 경고)
□ 맞춤법 검사
```

### 수빈 검수 (Tier 2만)
- Tier 1: 자동 검수 통과하면 바로 발행
- Tier 2: AI 초안 → 수빈 확인 → 수정 필요시 수정 → 발행
- Tier 3: 수빈 직접 작성 → AI 초안 보조 → 수빈 최종 확인

### 검수 대시보드 (수빈용)
- 오늘 자동 발행된 페이지 목록
- 검수 대기 중인 Tier 2 페이지
- 각 페이지 트래픽/성과 요약
- 문제 있는 페이지 경고 (평점 낮은 장소가 추천에 포함된 경우 등)

## STEP 5: 발행 (자동)

### 발행 프로세스
```
1. 콘텐츠 생성 완료
2. 자동 검수 통과
3. Next.js 빌드 트리거 (Vercel)
4. 정적 페이지 생성 (SSG)
5. 배포 (자동)
6. Google Search Console에 색인 요청
7. 네이버 서치어드바이저에 색인 요청
```

### 발행 스케줄
```
Tier 1 (자동): 매일 5~10페이지 (점진적 증가)
Tier 2 (감수): 주 3~5편
Tier 3 (직접): 월 2~4편

주의: 한 번에 100페이지 발행하면 구글이 스팸으로 판단할 수 있음.
→ 하루 최대 10페이지, 점진적으로 늘려감.
```

### 자동 업데이트 주기
```
장소 데이터: 주 1회 갱신 (영업 중단, 평점 변동 반영)
시즌 페이지: 매년 자동 갱신 (2026 벚꽃 → 2027 벚꽃)
키워드: 주 1회 트렌드 체크
```
